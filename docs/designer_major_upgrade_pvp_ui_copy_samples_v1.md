# The Defier《逆命者》V10.0 真 PVP UI 文案样例 V1

## 0. 文档定位

本文定义 V10 真 PVP 首版的玩家可见文案、按钮、toast、错误提示、断线/超时状态、结算页和复盘卡片样例。

阶段说明：本文只冻结玩家可见口径和浏览器验收要点，不冻结具体页面文件、测试文件或组件落点。以下消费方是当前建议映射，进入开发前必须按最新仓库结构重新确认。

建议消费方：

- `js/scenes/pvp-scene.js`
- `js/scenes/pvp-live-scene.js`
- `tests/browser_pvp_live_audit.mjs`
- `tests/browser_pvp_live_mobile_audit.mjs`
- `tests/sanity_intro_progress_sync_checks.cjs`

本文只给首版中文文案，不做多语言。

### 0.1 文档优先级与漂移说明

- 本文只冻结玩家可见口径、按钮、toast、审计断言和文案红线。
- 若规则、奖励、隐藏信息、社交边界或上线门槛与主方案冲突，以 `docs/designer_major_upgrade_overall_plan_v1.md` 为准。
- 本文中的 scene、service、test 路径只作为当前建议消费方，不作为已冻结开发落点。
- 进入开发前必须先完成文档漂移审计，再按最新仓库结构重生成实施计划。

| 模式 | 入口说明 | 是否计分 | 是否保留复盘 | 何时推荐 |

| 场景 | 必须触达 | 不允许 |
| --- | --- | --- |
| 回归登录 | 汇总本赛季影响排位的规则、合法池和奖励变化 | 直接弹长文阻断进入练习或回放 |

## 5. 玩家可见状态文案

本节默认冻结玩家可见口径，不直接冻结 wire enum。进入协议、fixture 或浏览器断言前，必须先确认 code 属于哪一类：

| 枚举面 | 例子 | 是否允许作为后端 intent reject wire code |
| --- | --- | --- |
| `intent_reject_code` | `not_in_match`、`match_not_active`、`not_your_turn`、`insufficient_energy` | 是 |
| `sync_state_code` | `action_pending`、`duplicate_action`、`sync_required`、`refresh_required`、`settling` | 否，只能作为状态同步或本地状态 |
| `transport_error_code` | `connection_lost`、`server_busy`、网络超时 | 否，只能作为传输或服务可用性提示 |
| `local_ux_hint_code` | `opponent_thinking`、`timer_low`、`emote_limited`、`emote_muted` | 否，只能由前端或 UI 状态机生成 |

### 5.1 Intent Reject 文案

| code | toast |
| --- | --- |
| `not_in_match` | 当前对局已失效。 |
| `match_not_active` | 现在还不能这么做。 |
| `not_your_turn` | 还未轮到你行动。 |
| `stale_state` | 局面已更新，正在同步服务端状态。 |
| `rule_version_mismatch` | 规则版本已更新，请刷新局面。 |
| `snapshot_locked` | 排位快照已锁定，不能在本局中改谱。 |
| `invalid_card` | 这张牌不在当前手牌中。 |
| `insufficient_energy` | 灵力不足，无法打出这张牌。 |
| `invalid_target` | 目标已失效，请重新选择。 |
| `duplicate_action_conflict` | 行动编号冲突，正在同步局面。 |
| `card_disabled` | 该内容不进入真人排位。 |
| `mulligan_used` | 本局已经调息换牌。 |
| `loadout_illegal` | 斗法谱未通过排位校验。 |
| `state_view_forbidden_field` | 状态同步异常，已停止公开。 |
| `action_timeout` | 已超时，服务端将按规则处理。 |
| `terminal_already_settled` | 对局已经结束。 |

### 5.2 同步、传输与本地提示文案

| code | toast |
| --- | --- |
| `loadout_version_mismatch` | 斗法谱版本已更新，请重新确认。 |
| `match_not_found` | 对局已结束或不存在。 |
| `connection_lost` | 连接中断，正在尝试恢复。 |
| `action_pending` | 行动已提交，等待服务端裁定。 |
| `duplicate_action` | 这次行动已经处理，正在恢复结果。 |
| `sync_required` | 本地局面已过期，正在同步服务端状态。 |
| `refresh_required` | 连接恢复失败，请刷新以获取权威局面。 |
| `action_confirmation_pending` | 上一操作待确认，请勿重复提交。 |
| `opponent_thinking` | 对手正在思考。 |
| `timer_low` | 剩余时间不多，请确认行动。 |
| `timer_expired` | 行动超时，服务端将按规则处理。 |
| `emote_limited` | 表情发送过快，请稍后再试。 |
| `emote_muted` | 已静音对手表情。 |
| `server_busy` | 天道裁定繁忙，请稍后重试。 |

预算命中不是 reject toast，必须以公开事件展示：

```text
护命预算拦截了 {preventedDamage} 点伤害，实际造成 {actualDamage} 点。
```

浏览器审计默认断言玩家可见文案，不直接断言 wire code。只有 `docs/designer_major_upgrade_overall_plan_v1.md` 和 `docs/designer_major_upgrade_requirements_v7.md` 同时声明为 machine contract 的 code，才允许进入协议级测试断言。

争议结论玩家文案：

| 结论 | 玩家文案 | 积分 / 奖励说明 |
| --- | --- | --- |
| `normal_result` | 复核完成：本局按规则正常结算。关键依据：{evidenceSummary}。 | 积分和奖励保持不变 |
| `invalidated_bug` | 复核完成：本局存在系统异常，已标记为无效局。 | 不改正式积分；可发非强度补偿 |
| `abuse_confirmed` | 复核完成：发现异常操作或刷分行为，已撤销异常收益。 | 撤销异常奖励；必要时触发冷却 |
| `insufficient_evidence` | 复核完成：有疑点但证据不足，暂不惩罚任一玩家。 | 保持结算或继续暂挂奖励 |
| `duplicate_report` | 该问题已处理，下面是已有结论摘要。 | 不产生新的积分或奖励变化 |

## 11. 文案红线

首版玩家可见文案禁止出现：

- 把残影称为正式排位对手。
- 把客户端胜负回执称为权威结算。
- 把护命预算描述成随机保护。
- 把无效局描述成玩家失败。
- 把断线判负描述成系统惩罚。
- 把再战描述成自动继续正式排位。
- 把战报分享描述成完整公开双方手牌。
- 暗示可以实时观战正式排位。
- 提供自由文本聊天入口。

推荐表达：

- `服务端裁定`
- `真人排位`
- `残影演武只作练习`
- `护命预算`
- `对局状态异常，未写入正式积分`
- `分享脱敏战报`
- `首版不开放实时观战`
