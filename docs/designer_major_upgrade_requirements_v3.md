# The Defier《逆命者》V6.0《天命裂界》功能需求文档 V3

## 1. 文档用途

本文是 [designer_major_upgrade_planning_v3.md](/Users/health/IdeaProjects/The-Defier/docs/designer_major_upgrade_planning_v3.md) 的配套需求文档。

上一份文档解决的是：

- 版本为什么要这样升级
- 方向应该往哪里走
- 整体产品想变成什么

本文解决的是：

- **V6.0 具体要做什么**
- **每个系统要新增哪些功能**
- **每个页面要补哪些信息和交互**
- **内容侧要新增哪些包**
- **哪些规则必须统一**

本文刻意**不写排期**、**不写人力分工**、**不写工时估算**，只聚焦“应该做成什么”。

## 2. V6.0 必做范围

V6.0 不应只是一个加卡包版本，而应包含以下 9 大交付板块：

1. **开局构筑层重做**
   - 角色二阶道途
   - 命格谱系
   - 灵契护道
2. **中局转折层重做**
   - 逆命誓约
   - 新节点类型
   - 章节内重大抉择
3. **战斗系统深化**
   - 命环路径树
   - 法则编织盘
   - 法宝炼器与器灵灌注
   - 章节世界规则
4. **主线章节重构**
   - 6 章化结构
   - 章节生态
   - 三幕 Boss
5. **内容包扩充**
   - 新角色
   - 新流派
   - 新卡牌
   - 新法宝
   - 新事件
   - 新敌人
6. **局外成长扩展**
   - 洞府经营
   - 传承 2.0
   - 图鉴 2.0
7. **长期留存系统**
   - 今日天机
   - 七日劫数
   - 众生试炼
   - 无尽深化
   - PvP 赛季深化
8. **界面与信息架构重构**
   - 主菜单层级
   - 战斗信息优先级
   - 高级系统视觉语言统一
9. **内容生产与验证基建**
   - 数据结构扩展
   - 内容模板化
   - 新门禁审计

如果上述任一大板块缺失，V6.0 的“大版本升级”感都会明显不足。

## 3. 玩家主流程需要重构成什么样

V6.0 的主流程应明确变成以下结构：

1. **主菜单**
   - 主线登天
   - 无尽轮回
   - 天道榜
   - 洞府
   - 藏经阁
2. **开局准备**
   - 选角色
   - 选道途
   - 选命格
   - 选灵契
3. **章节推进**
   - 看章节天象与地脉
   - 走地图节点
   - 中途形成构筑
4. **章节转折**
   - 触发誓约
   - 选择风险路线
   - 面对精英 / Boss 检定
5. **局内结算**
   - 章节奖励
   - 传承精魄
   - 图鉴解锁
   - 洞府进度
6. **局外回环**
   - 洞府建设
   - 传承解锁
   - 图鉴研究
   - 挑战任务
   - PvP / 无尽 / 周挑战切换

主流程的关键要求是：

- 玩家从第一分钟就知道这一局的“身份”
- 玩家在中局必须遇到至少一次真正改命的选择
- 玩家失败后仍有明确的局外收获
- 玩家完成一局后，自然被牵引去做下一件事

## 4. 开局系统需求

### 4.1 角色二阶道途

### 要做什么

现有 6 名角色全部升级为“双道途角色”，并新增 1 名解锁角色。

### 具体要求

每名角色都要有：

- `2` 条道途定位
- `1` 套基础起手风格差异
- `1` 组推荐命格方向
- `1` 组推荐灵契搭配
- `1` 组专属誓约偏好
- `1` 组角色专属事件

### 角色需求表

| 角色 | 道途 A | 道途 B | 主要玩法差异 |
|------|--------|--------|--------------|
| 林风 | 逆势压血 | 绝斩处决 | 压低生命后爆发 vs 破甲单点 |
| 香叶 | 回生续航 | 蚀命诅咒 | 治疗转生存 vs 治疗转伤害 |
| 无欲 | 功德防反 | 业火献祭 | 防御反击 vs 自损爆发 |
| 严寒 | 解析规划 | 因果延迟 | 资源掌控 vs 延时触发 |
| 墨尘 | 星律连携 | 时序预埋 | 节奏连段 vs 下一回合布局 |
| 宁玄 | 灵器共鸣 | 镇阵壁垒 | 法宝强化 vs 守阵消耗 |
| 沈夜白 | 咒契驱役 | 禁术噬命 | 诅咒积累 vs 代价换终局 |

### 完成状态

玩家在角色选择页，不只是选立绘和初始属性，而是在选一种“这局怎么玩”的入口。

### 4.2 命格谱系

### 要做什么

新增局内命格系统，作为角色与卡组之间的一层轻量构筑。

### 功能要求

- 开局给出 `3` 选 `1`
- 每章末允许强化一次
- 少量事件、洞府研究、周挑战可额外改写命格
- 图鉴中展示命格来源、适配角色和适配流派

### 命格类型要求

| 类型 | 目标体验 | 示例方向 |
|------|----------|----------|
| 爆发型 | 快速建立爽点 | 首击增伤、斩杀回能 |
| 续航型 | 稳定滚雪球 | 治疗转护盾、护盾留存 |
| 控制型 | 控节奏、拖关键回合 | 减益扩散、冻结推迟 |
| 赌博型 | 高风险高收益 | 自损抽牌、延迟爆发 |

### 内容要求

- 首发命格总量 `24`
- 每名角色至少 `4` 个高度契合命格
- 至少 `6` 个可跨角色泛用命格

### 界面要求

- 开局选择页要能清楚展示“命格会把这局导向什么”
- 命格详情必须显示升级后会发生什么
- 战斗中要能看到当前命格正在产生哪些效果

### 4.3 灵契护道

### 要做什么

新增灵契系统，作为 V6.0 的新核心卖点和局内局外双向成长目标。

### 功能要求

- 每局只能带 `1` 个灵契
- 灵契具备：
  - `1` 个常驻被动
  - `1` 个蓄能主动
  - `1-3` 个成长阶段
  - `1` 份图鉴故事
- 灵契可通过：
  - 主线事件
  - 洞府研究
  - 周挑战奖励
  - 无尽特殊掉落
  获得或强化

### 首发灵契需求

| 灵契 | 核心作用 |
|------|----------|
| 霜螭 | 延后敌方关键回合、冻结节奏 |
| 烛鸦 | 灼烧、献祭、死亡收益 |
| 星狐 | 抽牌、预知、牌序优化 |
| 玄龟 | 护盾、减伤、留存 |
| 魇蝶 | 诅咒、脆弱、减益扩散 |
| 灵猿 | 连击、回能、行动流 |
| 剑魄 | 单体破防与斩杀 |
| 器灵 | 法宝套装、装备触发强化 |

### 战斗表现要求

- 灵契被动必须在战斗 HUD 中有独立标识
- 主动技必须有蓄能条或冷却读条
- 灵契触发应有专属日志与特效

### 局外要求

- 图鉴可查看灵契故事、出处、成长效果
- 洞府内可查看灵契研究进度
- PvP 防守阵法允许挂接灵契

## 5. 中局转折系统需求

### 5.1 逆命誓约

### 要做什么

把章节间的普通奖励升级为真正影响整局走向的“誓约抉择”。

### 功能要求

- 第 `1`、`3`、`5` 章末出现誓约选择
- 单局最多持有 `2` 条誓约
- 同条誓约可升阶，但不能无限叠层
- 部分事件与禁术坛可提前接触誓约碎片

### 设计要求

每条誓约必须满足：

- 明确收益
- 明确代价
- 可读的战斗表现
- 至少有 `1` 条适配 build
- 至少有 `1` 个自然弱点

### 首发誓约表

| 誓约 | 核心收益 | 核心代价 |
|------|----------|----------|
| 焚命誓 | 爆发和抽牌上升 | 最大生命下降 |
| 镇狱誓 | 护盾与反击上升 | 治疗削弱 |
| 窥天誓 | 法则与事件收益更高 | 商店变贵 |
| 噬业誓 | 业障转中后期收益 | 前期稳定性下降 |
| 破界誓 | 章节机制和指令更强 | 资源更吃紧 |
| 归寂誓 | 弃牌、消耗、献祭体系强化 | 手牌容错下降 |
| 血月誓 | 压血阈值收益 | 续航难度提升 |
| 悬镜誓 | 复制与反照收益 | 高额自伤或反噬 |

### 界面要求

- 誓约选择页必须让玩家立刻看懂“这是在赌什么”
- 战斗内要有誓约状态条或悬浮详情
- 章节地图上要能提醒当前誓约会偏好哪些节点

### 5.2 新地图节点

### 要做什么

让节点不再只是功能分发，而成为路线决策本身。

### 必做节点

| 节点 | 作用 |
|------|------|
| 观星台 | 预览未来奖励、章节词缀、Boss 倾向 |
| 炼器坊 | 法宝重铸、器灵灌注、套装修正 |
| 禁术坛 | 用生命 / 诅咒 / 精神负担换高收益牌或誓约碎片 |
| 灵契窟 | 获得灵契或强化灵契能力 |
| 记忆裂隙 | 章节叙事与特殊抉择 |
| 试炼碑 | 主动叠加难度换取高稀有奖励 |

### 设计要求

- 每个新节点都要提供独立的策略价值
- 节点不应只是“资源多一点”
- 节点必须至少影响以下其中两项：
  - 卡组方向
  - 法宝方向
  - 法则方向
  - 誓约风险
  - 章节路线

### 5.3 事件系统升级

### 要做什么

把事件从“随机小收益”升级为“局内叙事 + 策略分歧”的核心承载点。

### 内容要求

- 新增事件 `28-32` 个
- 每章至少 `4` 个章节专属事件
- 每个角色至少 `2` 个专属事件
- 每个新流派至少 `2` 个定向事件

### 事件结构要求

事件至少覆盖以下 4 类：

- 构筑改向型
- 风险交易型
- 世界观叙事型
- 稀有掉落与解锁型

### 事件产出要求

事件不能只给金币或回血，应能产出：

- 命格升级
- 誓约碎片
- 灵契线索
- 临时法则
- 禁术牌
- 法宝重铸机会
- 章节捷径或绕路机会

### 5.4 商店和营地

### 商店要做什么

商店升级为三页结构：

| 页签 | 要做的内容 |
|------|------------|
| 基础页 | 卡牌、法宝、移除、常规服务 |
| 契约页 | 禁术、诅咒交易、誓约碎片、灵契材料 |
| 传闻页 | 未来章节提示、Boss 情报、流派偏向消息 |

### 商店交互要求

- 明确展示“贵但值”的高风险商品
- 显示当前誓约 / 命格 / 路径是否影响价格或商品池
- 给玩家看见“这一页是为你的 build 准备的”感觉

### 营地要做什么

营地升级为四项操作：

| 操作 | 功能 |
|------|------|
| 静养 | 回复与解除异常 |
| 冥想 | 命格、法则、命环节点强化 |
| 逆铸 | 卡牌分叉升级、法宝重铸 |
| 观星 | 预览下章生态和路径偏好 |

### 营地规则要求

- 不允许所有选项都无脑等价
- 玩家必须明显感到“我只能保一个方向”
- 部分角色、誓约和洞府研究可拓展营地操作上限

## 6. 战斗系统深化需求

### 6.1 命环路径树

### 要做什么

把现有命环路径扩展为真正的成长树，而不是单点被动。

### 功能要求

- 路径分主路径、碑文节点、跃迁节点
- 每条路径都能明确改变 build 方向
- 关键等级节点必须弹出选择，而不是自动线性成长

### 路径要求

| 路径 | 核心方向 |
|------|----------|
| convergence | 首段强、节奏快、回合顺 |
| resonance | 技能连锁、抽牌、回响 |
| destruction | 压血爆发、风险处决 |
| wisdom | 法则、事件、信息优势 |
| sanctum | 护盾、阵御、守转攻 |
| calamity | 诅咒、业障、侵蚀 |

### 交互要求

- 地图中可看到当前路径偏好节点
- 战斗中能看见路径已触发和待触发效果
- 图鉴中可查看路径树总览

### 6.2 法则编织盘

### 要做什么

把法则从“散装被动”升级为“可规划的组合系统”。

### 功能要求

- `1` 主法则槽
- `2` 副法则槽
- `1` 禁术槽
- 支持编织共鸣、相克和禁忌组合

### 组合规则要求

- 主法则决定战斗主语法
- 副法则用于支撑、补短板、做反制
- 禁术必须高收益高代价
- 编织盘要显示当前组合倾向和风险

### 产出要求

法则获取不再只靠单一掉落，应来自：

- 盗取
- 事件
- 誓约
- 商店契约页
- 洞府研究

### 6.3 法宝炼器与器灵灌注

### 要做什么

让法宝从单次掉落品变成可经营、可深化的中层构筑系统。

### 功能要求

- 保留现有法宝品质规则
- 新增炼器重铸
- 新增器灵灌注
- 新增套装二段和三段效果

### 系统要求

- 法宝效果应优先改变玩法而非单纯堆数值
- 法宝图鉴要能展示套装关系、来源、适配流派
- 器灵灌注必须服务少量核心法宝，不做所有法宝都可灌注

### 内容要求

- 新增法宝 `20-24`
- 新增至少 `4` 套完整套装
- 新增至少 `6` 件“核心件”
- 新增至少 `8` 件“形态件”

### 6.4 战场系统：天象、地脉、阵面

### 要做什么

把章节环境正式做成会改变打法的“世界规则”。

### 三层规则

| 层级 | 作用 |
|------|------|
| 天象 | 战斗内动态变化 |
| 地脉 | 该章持续规则 |
| 阵面 | 敌我站场关系和阵列影响 |

### 功能要求

- 每章至少 `1` 套天象
- 每章至少 `1` 条地脉
- 至少 `3` 章具备明显阵面差异
- 战斗 UI 必须能清楚展示当前世界规则

### 设计要求

- 世界规则要改变打法，但不能让特定 archetype 直接废掉
- 章节规则要先教后考，不应第一次出现就直接秒人

### 6.5 战斗可读性

### 要做什么

因为新系统会变多，所以战斗界面必须重新定义优先级。

### 战斗 HUD 必须增加的信息

- 当前命格
- 当前誓约
- 当前灵契
- 当前章节天象 / 地脉
- 当前法则编织组合
- 当前法宝套装激活数

### 优先级要求

战斗内最重要的信息必须按以下顺序呈现：

1. 敌方下一回合意图
2. 我方可用资源
3. 本回合关键加成与风险
4. 中层系统状态
5. 详细日志与历史记录

### 视觉要求

- 高级系统统一用命纹图形、颜色、边框规则表达
- 少用纯文本堆说明
- 保留移动端可读性，避免浮层互相遮挡

## 7. 内容包需求

### 7.1 卡牌内容包

### 要做什么

把卡牌内容扩展到足以支撑“大版本新鲜感”的级别。

### 内容量要求

- 总卡量从 `178` 提升到 `230-240`
- 新增两套主 archetype：
  - `cursebound`
  - `soulforge`
- 强化现有 `6` 套核心流派
- 新增桥接卡 `12`
- 新增中立功能卡 `8-12`

### 设计要求

- 每个流派都有：
  - 起手牌
  - 中轴牌
  - 终结牌
  - 桥接牌
  - 应对型牌
- 每套流派都能说明“靠什么赢”

### 分叉升级要求

- 关键卡不只做数值 +，要做功能分支
- 关键升级路线应体现不同 build 方向

### 7.2 敌人与生态内容包

### 要做什么

增加新敌人与新生态模板，让章节不再像“换皮敌人库”。

### 内容量要求

- 新增敌人 `18-24`
- 每章至少 `1` 组生态模板
- 每章至少 `1` 组精英组合
- 每章至少 `1` 套 Boss 三幕机制

### 生态要求

每章敌人组合必须能形成：

- 输出压力
- 控制压力
- 资源压力
- 构筑针对压力

### AI 要求

- 敌方意图要更清晰
- 精英和 Boss 要能读玩家状态做轻度应对
- 不追求完全智能，但要避免机械重复

### 7.3 Boss 内容包

### 要做什么

所有大章 Boss 都应具备鲜明身份、转阶段和破局点。

### 必做要求

- 开场立场机制
- 中场转阶段
- 终局大招
- 至少 `1` 个反制窗口
- 至少 `1` 个视觉记忆点

### 破局设计要求

Boss 不能只靠高血高攻体现强度，应加入：

- 阵眼拆解
- 状态打断
- 条件爆发
- 资源交换
- 站位破坏

### 7.4 事件与叙事内容包

### 要做什么

让“剧情”与“玩法改变”强绑定。

### 内容要求

- 每章至少 `1` 条连续性叙事线
- 每个灵契至少 `1` 条获取故事
- 新角色至少 `1` 条解锁线
- 终章必须有与誓约 / 命环 / 法则相关的总回收

### 表现要求

- 事件文本应更有命运感和仙侠气质
- 重大抉择要有独立 UI 气氛
- 不把复杂效果都塞在长文案里，要拆成效果摘要

## 8. 局外成长与留存需求

### 8.1 洞府经营

### 要做什么

新增局外中枢系统，承接失败后的成长目标和版本长期运营目标。

### 洞府房间要求

| 房间 | 要做的内容 |
|------|------------|
| 藏经阁 | 图鉴研究、构筑快照、Boss 情报 |
| 炼器室 | 法宝研究、套装解锁、器灵灌注入口 |
| 观星台 | 周挑战、Seed 回放、章节预兆 |
| 伏魔台 | 高难试炼、Boss 记忆战、特殊奖励 |

### 洞府规则要求

- 洞府成长应偏解锁，不做面板膨胀
- 洞府研究结果应明确反哺主线、无尽、PvP 或图鉴

### 8.2 传承系统 2.0

### 要做什么

让传承继续存在，但更多承担“打开新玩法”的作用。

### 解锁方向

- 新命格池
- 新灵契
- 新章节事件
- 新法则槽位或编织研究
- 新营地服务
- 新挑战规则
- 新外观与称号

### 要避免什么

- 不做大量永久伤害百分比
- 不做无限制生命值膨胀
- 不做“后期老玩家面板碾压新玩家”的纯数值成长

### 8.3 图鉴 2.0

### 要做什么

图鉴要升级为研究与复盘工具。

### 必做页面

- 卡牌图鉴
- 法宝图鉴
- 法则图鉴
- 灵契图鉴
- 章节 / 敌人图鉴
- Boss 档案
- 构筑快照
- 个人高光记录

### 功能要求

- 显示来源
- 显示适配流派
- 显示未解锁线索
- 显示相关联条目
- 支持搜索和筛选

### 8.4 周挑战体系

### 要做什么

把局内内容转成可重复消费的留存玩法。

### 必做模式

| 模式 | 要做的内容 |
|------|------------|
| 今日天机 | 每日短挑战，指定角色 / 章节 / 词缀 |
| 七日劫数 | 周积分挑战，可重复冲分 |
| 众生试炼 | 全服统一规则挑战，附排行榜 |

### 奖励要求

- 以称号、外观、图鉴、洞府材料、天机印为主
- 少给纯数值型成长

### 8.5 无尽与 PvP 深化

### 无尽要做什么

- 小轮回 / 大轮回结构
- 轮回偏执词缀
- 世界裂隙事件
- 无尽专属图鉴
- 无尽专属荣耀线

### PvP 要做什么

- 赛季公共法则
- 防守阵灵接入
- 镜像回放
- 赛季道印奖励
- 周论道合约规则池

### 目标要求

- 无尽负责“构筑极限”
- PvP 负责“阵法博弈与展示”
- 两者不能做成纯数值重复劳动

## 11. 文案、世界观与包装需求

### 11.1 世界观回收

### 要做什么

让命环、法则、誓约、灵契、章节、洞府在世界观里形成同一套话语体系。

### 文案要求

- 命环负责“命数与改命”
- 法则负责“世界规则与夺取”
- 誓约负责“代价与执念”
- 灵契负责“同行与见证”
- 洞府负责“传承与归宿”

### 11.2 版本包装

### 要做什么

把 V6.0 做成真正有主题感的资料片级版本。

### 包装要求

- 版本名：`天命裂界`
- 版本主视觉：裂开的命环、穿透天幕的法则裂缝、灵契环绕
- 版本标语要突出“逆天改命进入下一阶段”

## 12. V6.0 完成后的应有状态

V6.0 完成后，玩家对《逆命者》的感受应出现以下变化：

- 以前是“这游戏内容挺多”
- 现在应变成“这游戏每一局都真不一样”

- 以前是“我在拿牌通关”
- 现在应变成“我在选择一条命去赌”

- 以前是“失败了，再开一把”
- 现在应变成“失败了，但我解锁了新命格、新灵契、新研究方向”

- 以前是“主线、无尽、PvP 有点分散”
- 现在应变成“我每次上线都有明确目标，而且这些模式是互相喂养的”

如果 V6.0 能达到以上状态，这次升级就不只是“加内容”，而是真正完成了一次产品层级的跃迁。
