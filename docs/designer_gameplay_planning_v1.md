# The Defier 玩法扩展策划案 V1（Designer 驱动）

## 1. 本轮策划输入
- 使用的技能方法：
  - `game-design-theory`：MDA、核心循环、难度曲线。
  - `card-game-design`：卡牌曲线、反模式、平衡校验。
  - `level-design`：地图节奏、节点流量、风险休整交替。
  - `economic-systems`：货币产消、商店定价、补给压力。
  - 本地 `game-designer`：命环世界观约束与五行语义。
- 现有实现边界（已核对代码）：
  - 卡牌/效果系统已支持 `applyBleed/applyMark/blockBurst/execute/mulligan/stance/...`。
  - 敌人模式已支持 `attack/multiAttack/debuff/heal/summon/multiAction`。
  - 已有流派：`hemorrhage/precision/entropy/bulwark`。
  - 已有系统：无尽压力、事件偏置、战场指令、敌人战术队列。

## 2. 版本目标（V1~V3）
1. 提升同一天域敌人的差异性，避免“数值换皮”。
2. 扩展构筑层深度：命环路径、法宝、卡牌三轴联动。
3. 让无尽模式形成“可规划的高压生态”，不是单纯数值堆叠。
4. 在每次扩展后保持全量自动化测试可通过。

## 3. 设计支柱
1. 清晰反馈：每个关键机制都要有战斗日志与 UI 提示。
2. 可反制：强机制必须给对策，不做无解机制。
3. 渐进复杂：前期易理解，中后期增加组合密度。
4. 有代价的强力：高收益动作必须绑定明确成本。

## 4. 系统扩展蓝图

### 4.1 命环路径扩展（玩法身份）
1. 新增路径专精被动层（Path Doctrine）：
   - `convergence`：首回合战场指令减耗 + 汇流计数收益。
   - `resonance`：技能牌触发后获得短回合护盾或抽牌。
   - `destruction`：低护盾时攻击增益，但恢复效率降低。
   - `wisdom`：事件收益偏重“知识/法则/临时规划”。
2. 路径与地图节点联动：
   - 每条路径定义“偏好节点 + 反感节点”微调权重。
   - 每 3 层触发一次路径小事件（可选收益分支）。
3. 路径成长里程碑：
   - 在命环等级 4/7/10 给出一次路径强化选择，避免线性加成。

### 4.2 法宝系统扩展（构筑第二引擎）
1. 新增法宝套装标签（2 件、3 件）：
   - `玄甲套`：护盾留存 + 反击伤害。
   - `裂脉套`：流血层转化为斩杀阈值增益。
   - `星衡套`：回合节奏类收益（抽牌/灵力平衡）。
2. 新增法宝品质内约束：
   - 神话级仍保持最多 1 件，不打破现有上限。
   - 套装强度优先做“机制增益”而非纯倍数。
3. 法宝来源扩展：
   - 事件、商店、营地强化、无尽赐福都可提供法宝片段或重铸机会。

### 4.3 卡牌与流派扩展（核心可玩性）
1. 新增 2 套流派（每套 15-18 张）：
   - `stormcraft`（雷策）：多段攻击 + 易伤管理 + 连击窗口。
   - `vitalweave`（回生）：治疗转护盾、治疗转伤害、续航换爆发。
2. 跨流派桥接牌：
   - 每套至少 4 张“桥接卡”，降低构筑锁死问题。
3. 新关键词策略：
   - 仅使用现有效果类型组合，不引入高风险全新底层关键字。
4. 卡牌平衡红线（来自 card-game-design 校验）：
   - 新卡默认不做 0 费无条件高价值。
   - 不做“任意检索”与“即时胜利”。
   - 重复抽牌引擎必须有费用或次数限制。

### 4.4 怪物生态扩展（解决同质化）
1. 每个天域配置 2 个“怪物族群模板”：
   - 例：压制型（debuff）、护阵型（defend+counter）、猎杀型（multiAttack）。
2. 普通怪引入“行为身份标签”：
   - `striker/guardian/hexer/supporter/disruptor`。
3. 同一天域差异化规则：
   - 至少保证 1 个“高压输出怪”、1 个“状态控制怪”、1 个“节奏干扰怪”。
4. 精英/Boss 扩展：
   - 每个阶段 Boss 增加 1 条可读“转阶段机制”提示与反制窗。

### 4.5 地图节点、事件、商店、营地扩展
1. 事件：
   - 每个流派新增 3-4 个专属事件，总计新增 16+。
   - 增加“短期收益 vs 长期代价”型选项。
2. 商店：
   - 新增“本局限时契约页”，提供高风险高收益交易。
   - 引入动态折扣：与无尽压力、路径、法宝共振相关。
3. 营地：
   - 增加“重配战场指令”、“法宝重铸”、“命环冥想”三类操作。
4. 试炼与锻炉：
   - 追加主题试炼词条与锻炉专精分支，提升路线规划价值。

### 4.6 无尽模式深化（长期留存核心）
1. 周期结构：
   - 13 周期为一轮回，3/6/9/12 周期触发阶段挑战。
2. 压力系统升级：
   - 压力影响敌人行为池、商店定价、事件权重、战场指令资源上限。
3. 赐福系统升级：
   - 新增“对冲型赐福”（降低压力但削弱奖励）与“赌命型赐福”。
4. 轮回专属 Boss 词缀：
   - 从现有词缀体系扩展出无尽专属组合词缀，避免中后期同构。

## 5. 开发分期（可直接执行）

### P1（怪物差异化周）
1. 扩充每个天域怪物族群模板与战术标签。
2. 为普通怪补齐 `multiAction/debuff/defend` 组合差异。
3. 新增怪物同质化审计测试（统计动作分布、标签覆盖）。

### P2（流派与卡牌周）
1. 上线 `stormcraft` 与 `vitalweave` 两套卡池。
2. 打通事件偏置、奖励偏置、命环路径偏置。
3. 新增卡牌风险校验脚本（0费/抽牌引擎/检索检测）。

### P3（地图与经济周）
1. 扩展事件、商店、营地、试炼节点。
2. 打通法宝套装与重铸路径。
3. 新增经济曲线测试（灵石产消、治疗与伤害兑换效率）。

### P4（无尽深化周）
1. 新增轮回阶段挑战与专属词缀。
2. 压力-收益-风险三角再平衡。
3. 强化无尽自动化回归与压力边界测试。

## 6. 测试与验收标准
1. Node 测试：
   - 新增 `sanity_enemy_ecology_diversity_checks.js`
   - 新增 `sanity_card_design_guardrail_checks.js`
   - 新增 `sanity_endless_phase_boss_checks.js`
2. Browser 审计：
   - 校验新事件分支可达、战场指令/词缀提示可读、怪物标签显示正确。
3. 手工验证：
   - 主流程：主菜单 -> 新游戏 -> 地图 -> 战斗 -> 奖励 -> 存档/读档。
4. 可发布门槛：
   - `tests/run_node_checks.sh` 全通过。
   - 全部浏览器审计脚本通过。
   - 至少 2 轮 Playwright 截图巡检通过。

## 7. 下一步执行策略
1. 先做 P1（怪物同质化治理），因为这是当前可玩性短板的主矛盾。
2. 然后做 P2（新增两套流派），快速提升新鲜感与构筑差异。
3. 每个阶段完成后立刻补齐自动化测试，避免后期集中返工。
