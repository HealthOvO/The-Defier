# The Defier《逆命者》V8.0《观星归卷》功能需求文档 V5

## 1. 文档用途

本文是 [designer_major_upgrade_planning_v6.md](/C:/workspace/The-Defier/docs/designer_major_upgrade_planning_v6.md) 的配套需求文档。

本文不写排期、不写工时、不写人力，只回答一件事：

> **V8.0《观星归卷》到底要做什么，做到什么程度，页面和系统要补哪些能力。**

## 2. V8.0 必做范围

`V8.0` 至少要交付以下 `9` 大板块：

1. **修行课题卡系统**
2. **章节答卷系统**
3. **远征作答面板**
4. **观星回响总结卡**
5. **答卷评级与偏题诊断**
6. **训练建议回流**
7. **归卷档案 / 修行书架**
8. **挑战 / PVP 结业验证联动**
9. **数据结构与自动化门禁扩展**

缺任何一块，`V8.0` 的“大闭环”都会不完整。

## 2.1 首发范围（建议）

首发建议优先承诺以下模块：

1. 修行课题卡系统
2. 章节答卷系统
3. 远征作答面板
4. 观星回响总结卡
5. 答卷评级与偏题诊断
6. 训练建议回流

这 `6` 块已经足以闭合出完整主体验：

`选题 -> 开章定标 -> 章内作答 -> 即时偏题反馈 -> 章节归卷 -> 下次训练建议`

## 2.2 第二阶段预留

建议第二阶段再继续放大以下模块：

1. 归卷档案 / 修行书架的深筛与长周期聚合
2. 挑战 / PVP 结业验证联动的完整跨模式证明链

原因是这两块更偏长期价值放大，而不是让 `V8.0` 主闭环成立的第一优先项。

## 3. 玩家主流程需要升级成什么样

`V8.0` 后，玩家的主流程需要演进为：

1. **课题选择**
   - 在观察站或 PVP 焦点档案中选定一份主修课题
   - 阅读本章 / 本轮的训练目标、建议路线与结业条件
2. **进入章节**
   - 查看章节答卷
   - 明确三条本章核心目标
   - 根据答卷锁定或微调推荐路线
3. **远征作答**
   - 在主线 / 支线推进中持续观察贴题度
   - 处理偏题提示、风险提示与关键节点顺序
4. **章节归卷**
   - 在结算页看到观星回响总结卡
   - 理解本章答卷完成度、偏题原因与实际回报
5. **训练回流**
   - 将本章结果转成新的训练建议
   - 重新回到观察站或归档书架继续准备下一次作答
6. **结业验证**
   - 把练过的课题投向挑战验证或 PVP 焦点验证
   - 获得“课题已完成”的证明与高价值归档

## 4. 修行课题卡系统

### 4.1 要做什么

新增“修行课题卡”，把样本、对手档案和训练目标压成一个可执行的主修目标。

### 4.2 功能要求

课题卡首发至少支持以下来源：

- 观察站精选命盘
- 观察站历史留痕
- PVP 焦点对手档案

每张课题卡至少包含：

- 课题标题
- 来源标题
- 训练标签
- 训练目标 `2-3` 条
- 推荐路线
- 推荐节点
- 资源预留建议
- 结业条件

### 4.3 行为要求

1. 玩家每次只能激活 `1` 张主修课题。
2. 允许切换课题，但切换后必须保留旧课题历史。
3. 课题卡必须能被：
   - 设为本章答卷来源
   - 设为长期练习方向
   - 从归档中再次启用

### 4.4 UI 要求

课题卡必须在以下地方可见：

- 观察站
- 章节进入前摘要
- 远征作答面板
- 章节结算总结卡

## 5. 章节答卷系统

### 5.1 要做什么

在章节开始时把当前课题转成一张本章答卷。

### 5.2 功能要求

每张答卷至少包含三类目标：

- 路线目标
- 执行目标
- 资源目标

每类目标都必须有：

- 目标描述
- 当前状态
- 完成条件
- 偏离判定条件

### 5.3 状态要求

每条目标至少支持以下状态：

- 未开始
- 进行中
- 已完成
- 已偏离

### 5.4 章节进入前展示要求

玩家进入章节前必须能看到：

- 当前课题来源
- 当前答卷三条目标
- 推荐路线
- 推荐节点
- 本章主提醒

## 6. 远征作答面板

### 6.1 要做什么

在现有远征面板中加入“答卷进度”层，让玩家章内持续知道自己在不在题上。

### 6.2 功能要求

远征作答面板至少要显示：

- 当前课题名
- 当前答卷目标
- 当前贴题状态
- 当前偏题提示
- 下一条建议动作

### 6.3 交互要求

1. 玩家点击推荐路线、一键锁线、承接悬赏、推进关键节点后，答卷状态必须同步刷新。
2. 偏题后不应直接判定失败，而应优先给出纠偏提示。
3. 允许“偏题但成功通关”，结算时再集中说明后果。

### 6.4 可读性要求

1. 章内不允许一次展示超过 `3` 条主目标。
2. 不允许同时显示多条高噪音警告。
3. 移动端不得出现横向滚动或布局挤爆。

## 7. 观星回响总结卡

### 7.1 要做什么

在章节结算页 / 奖励页新增版本级主卡“观星回响总结卡”。

### 7.2 功能要求

总结卡必须输出：

- 本章课题来源
- 实际路线
- 完成目标
- 偏离目标
- 样本回响收益
- 答卷评级
- 下一次训练建议

### 7.3 文案要求

总结卡必须用玩家能读懂的自然语言回答：

- 我这章读的是什么题
- 我这章怎么答的
- 我哪里答对了
- 我哪里答偏了
- 这份样本最终值不值

### 7.4 奖励页联动要求

奖励页必须显示：

- 当前奖励补的是哪类短板
- 与当前课题是否同向
- 更适合立刻兑现还是下一章使用

## 8. 答卷评级与偏题诊断

### 8.1 要做什么

新增一套“答卷质量”评价系统。

### 8.2 功能要求

首发至少要有以下评分维度：

- 路线贴合
- 执行稳定
- 风险处理
- 收束质量

首发至少要有以下结果标签：

- 稳答
- 险答
- 偏答
- 超纲

### 8.3 诊断要求

偏题诊断至少要覆盖：

- 路线偏离
- 节点顺序错误
- 高压资源使用过早
- 风险判断失误

### 8.4 解释要求

任何评级结论都必须带说明文本，避免黑箱评分。

## 9. 训练建议回流

### 9.1 要做什么

让章节结果自动转成下一次训练建议。

### 9.2 功能要求

每次结算至少生成 `1` 条主建议，建议类型至少包括：

- 路线训练
- 资源训练
- 节奏训练
- 风险处理训练

### 9.3 回流入口要求

训练建议必须能回流到：

- 观察站
- 归卷档案
- 下一章答卷预览

### 9.4 失败价值化要求

失败结算不能只给失败结果，必须至少给出：

- 主要偏题原因
- 一条可执行建议
- 一条可复现的下一次训练方向

## 10. 归卷档案 / 修行书架

### 10.1 要做什么

把章节答卷历史与观察站样本历史整合成“修行书架”。

### 10.2 功能要求

归档至少支持以下筛查维度：

- 课题主题
- 章节
- 路线
- 评级
- 偏题原因
- 结局

### 10.3 阅读视角要求

书架至少提供以下视角：

- 按课题看成长
- 按章节看作答史
- 按偏题原因看训练缺口

### 10.4 操作要求

玩家应能直接：

- 回看一份归卷
- 重新设为当前课题
- 与最佳样本对比

## 11. 挑战 / PVP 结业验证联动

### 11.1 要做什么

让玩家把当前主修课题投向真正的“结业验证点”。

### 11.2 挑战验证要求

挑战验证至少要支持：

- 使用当前课题生成一条验证目标
- 验证完成后生成结业记录
- 结业记录进入归档书架

### 11.3 PVP 验证要求

PVP 焦点验证至少要支持：

- 使用焦点对手 dossier 生成课题
- 对局后判断是否满足课题目标
- 将结果回写到课题历史

### 11.4 价值要求

结业验证不能只是加一层标记，必须至少影响：

- 归档权重
- 课题完成状态
- 高质量样本展示

## 12. UI / 视觉要求

### 12.1 章前页

要新增：

- 课题摘要区
- 答卷目标区
- 推荐路线与节点区

### 12.2 远征页

要新增：

- 作答进度区
- 贴题 / 偏题状态
- 下一步建议提示

### 12.3 结算页 / 奖励页

要新增：

- 观星回响总结卡
- 答卷评级区
- 偏题诊断区
- 下一次训练建议区

### 12.4 移动端要求

必须确保：

- 两步内看到课题核心信息
- 总结卡不爆布局
- 作答面板不横向溢出

## 13. 数据结构与自动化门禁

### 13.1 数据层必须新增

至少新增或扩展以下结构：

- `training_topic_cards`
- `chapter_answer_sheet`
- `answer_sheet_rating`
- `answer_sheet_deviation_reasons`
- `training_recommendations`
- `cultivation_bookshelf_entries`

### 13.2 自动化门禁必须新增

至少增加以下检查：

- 课题卡生成与切换
- 章节答卷生成与状态推进
- 远征答卷面板刷新
- 章节结算总结卡渲染
- 答卷评级与偏题诊断输出
- 训练建议回流
- 归卷档案筛查
- 挑战 / PVP 结业验证写回

## 14. 最终结论

`V8.0《观星归卷》` 的核心，不是再给玩家更多面板，而是：

- 让每一章真的像一份题目
- 让每一局真的留下作答痕迹
- 让每一次结算真的能交卷并复盘
- 让 challenge、observatory、expedition、pvp 首次围绕同一条修行主线运作

如果 `V7.0` 解决的是“系统之间开始互相看见”，那么 `V8.0` 要完成的是：

> **让这些系统第一次像同一条修行之路的不同阶段，而不是一组并排很强的模块。**
