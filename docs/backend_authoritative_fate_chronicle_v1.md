# S111 权威命途长卷与三证归卷 V1

## 1. 目标

S111 继续收口两条长期缺口：

1. 现有服务端权威战斗仍主要位于赛季司的独立试炼切片，主菜单缺少一条不依赖同时在线、可持续推进的后端主线。
2. 众生试炼、世界裂隙、真人论道和同道远征已有可信结果，但玩家仍要在多个入口自行判断本周该做什么。

本阶段继续沿用“命途长卷”与“三证归卷”协议面，并同步 S111 锁定内容：

- 命途长卷仍是三章、服务器全权裁定的单人篇章，但当前 v2 新周卷改为每章三条实际誓约；
- 三证归卷把五类权威结果折叠为每周任选路线，不增加每日签到或连胜压力；
- 所有经济奖励只发 `renown`，`rewardImpact=cosmetic_only`、`powerImpact=none`。

## 2. 玩家循环

### 2.1 命途长卷

1. 玩家从主菜单进入“命途长卷”，无需先加入队伍或等待真人。
2. 每周长卷包含三章，按第一章、第二章、终章顺序解锁。
3. 当前 v2 新周卷中，每章提供稳健、进取和处置三条誓约。誓约绑定不同标准牌组、生命、战斗长度和敌人组合，不读取本地主线牌组、法宝或永久战力。
4. 第三条处置誓约分别是第一章“定稿誓”、第二章“审镜誓”、第三章“封卷誓”；选中该誓约发车后，权威 run 会在章节中段出现二选一。选择只改变后续敌人池、路线合同和构筑奖励偏向，不提供永久战力，也不改变协议命令面。
5. 玩家选定誓约后，服务端生成固定周种子并创建绑定的 `fate_chronicle` 权威 run；处置誓约的章中选择继续复用该 run 的 `select_node` 命令。
6. 选路、出牌、结束回合、奖励选择、放弃、快照恢复和完整重放继续复用 authoritative-runs V2。
7. 通关后由长卷服务读取完整重放回执，原子投影章节完成度、最佳成绩和解锁状态。
8. 失败、放弃或过期不会扣次数、不会倒退已完成章节，可立即重试；同一章同一实际誓约在一周内使用同一规则和种子，允许玩家学习和精进。
9. 完成任一实际誓约即可解锁下一章；完成本章全部实际誓约时记为“全誓”，只影响档案展示与纯外观里程碑。
10. 旧 v1 已冻结周卷继续保持每章两誓约快照；创建于旧周卷的 run 和档案不会在 S111 后被强制扩展为九誓约。

### 2.2 三证归卷

每周只读取以下五类 `server_authoritative` 完成事实：

| 凭证 | 权威模式 | 成立条件 |
| --- | --- | --- |
| 长卷证 | `fate_chronicle` | 任一章节形成完整重放结算回执 |
| 众生证 | `challenge_ladder` | 正式尝试形成权威结果 |
| 裂隙证 | `world_rift` | 正式出征完成并形成权威贡献 |
| 论道证 | `pvp_live` | 正式真人排位完成结算；胜负都计，友谊赛不计 |
| 同道证 | `relay_expedition` | 本人完成一棒并形成权威回执 |

归卷阈值：

- `0-1/5`：未归卷；
- `2/5`：基础归卷，可领取本周唯一经济奖励；
- 五类权威活动任取两类即可基础归卷，不强制参与 PVP；
- `3/5`：升格，仅升级本周档案等级；
- `4/5`：辉卷，仅展示荣耀；
- `5/5`：全证，仅展示荣耀。

基础归卷奖励为 120 荣誉。3-5 证不再增加货币，避免把自由选择重新变成全勤任务。

## 3. 公平与低压力合同

- 不存在每日登录、连续登录、每日首胜、连胜或必须清空全部模式的要求。
- 玩家只需任意两类凭证即可拿到本周全部经济收益。
- PVP 只要求正式完成，不要求胜利；平衡和匹配仍由 live PVP 自身规则负责。
- 社交玩法有凭证价值，但不是归卷前置；长卷、众生试炼、世界裂隙、真人论道中任意两类即可独立完成。
- 旧本地 PVE、旧挑战、旧远征、镜像练习、友谊赛、客户端自报分数和 `client_observed/server_verified` 事件都不能铸证。
- 命途长卷不读取账号永久牌组、属性、法宝、存档资源或付费内容；所有玩家面对相同目录、相同誓约规则和同周种子。
- 长卷不设置正式榜，不把通关速度变成必须竞争的奖励来源。

## 4. 命途长卷内容

协议名继续为 `authoritative-fate-chronicle-v1`。authoritative run schema 与 commands 不变，当前内容版本提升为 `authoritative-trials-v7`，fate 目录保持 `fate-chronicle-catalog-v2 / fate-chronicle-rotation-v2`。S112 在 v7 中加入敌意反制题面并修复敌方格挡生命周期；该规则只由 run 创建时冻结的内容快照启用，v1-v6 旧 run 不回填。S109 为每站提供两条服务器权威路线合同，公开风险档、敌方压力、奖励档和通关路线分，但不下发内部倍率；稳进、争衡、险锋分别形成容错、压力与回报递增的真实取舍。旧 v1 冻结周卷按创建时的 catalog/rotation 快照完整回放，不会被当前 v2 新周卷或后续合同重算。S108 已在不扩大战斗效果语法的前提下，把每战奖励升级为两张互异卡牌报价、服务器指定的卡牌精修，以及按生命和牌组状态动态出现的调息、固本或裁牌选择。

卡牌精修只覆盖基础数值牌和高费终结牌；观微、裂隙、流炁等滤抽、倍率与能量核心不提供直接升级。裁牌只允许移除破势或守心，第二战后才开放、每局最多两次，并始终保留至少两张直接伤害牌、两张格挡牌和八张总牌，避免构筑进入不可玩的软锁状态。

### 第一章：照火问心

- `chronicle-ember-guard`：稳健誓约，3 战，较高生命与护盾牌；
- `chronicle-ember-edge`：进取誓约，3 战，较低生命与进攻/易伤牌。
- `chronicle-ember-proof`：定稿誓，中段二选一后改写后续敌人池、路线合同与构筑奖励偏向。

### 第二章：镜命辨真

- `chronicle-mirror-guard`：稳健誓约，4 战，强调回复与抽滤；
- `chronicle-mirror-edge`：进取誓约，4 战，有总回合预算，强调主动收束。
- `chronicle-mirror-audit`：审镜誓，中段二选一后改写后续敌人池、路线合同与构筑奖励偏向。

### 第三章：裂天归卷

- `chronicle-rift-guard`：稳健誓约，5 战，战间小幅回复；
- `chronicle-rift-edge`：进取誓约，5 战，低容错、高计分倍率。
- `chronicle-rift-seal`：封卷誓，中段二选一后改写后续敌人池、路线合同与构筑奖励偏向。

首个敌意必须低于满血，任一誓约都不能出现玩家尚未行动即被击杀。三类誓约的目标是改变决策节奏、敌人分布和构筑偏向，不是制造一条显著支配路线，也不是注入永久成长。

## 5. 时间与恢复

- 周期以 UTC 周一 00:00 开始，持续 7 天；另有 7 天只读领奖窗口。
- 新 run 只能在当前周创建；已创建 run 最长可恢复 72 小时，但不能跨过该周结算宽限继续投影。
- 归属以服务端创建时间和权威回执时间为准，不读取客户端时间。
- 同账号同一时刻最多有一条 active 长卷 run。
- `reserved` 发车失败可恢复；绑定失败必须作废孤儿 run 并允许重试。
- current 读取、结果提交和奖励领取都会先恢复“权威 run 已终态、业务投影未完成”的窗口。
- 所有截止判断必须在取得 SQLite 写锁后重新读取服务端时间。

## 6. 数据模型

迁移 `0011_authoritative_fate_chronicle` 新增：

### 命途长卷

- `fate_chronicle_rotations`：不可变周目录快照、catalog/rule version、时间边界、章节/誓约规则和哈希；
- `fate_chronicle_attempts`：账号、轮换、章节、誓约、clientAttempt、mutation、run 绑定和状态；
- `fate_chronicle_results`：不可变权威回执投影、摘要、成绩和完整性引用；
- `fate_chronicle_progress`：账号每轮每章的完成誓约集合、最佳结果、全誓投影与解锁投影；
- `fate_chronicle_reward_claims`：章节/全誓/全卷纯外观荣誉领取事实；
- `fate_chronicle_mutations`：start/submit/claim 的请求哈希和脱敏回执；
- `fate_chronicle_ops_events/counters`：脱敏事件与聚合计数。

### 三证归卷

- `weekly_archive_cycles`：不可变周规则快照和哈希；
- `weekly_archive_reward_claims`：基础归卷唯一领奖事实；
- `weekly_archive_mutations`：领奖幂等回执；
- `weekly_archive_ops_events/counters`：脱敏归卷事件和聚合计数。

凭证事实不复制客户端状态。读取时从 `progression_events` 中筛选五个允许模式、`trust_tier=server_authoritative`、`activity_completions>0`，按周和模式去重。业务 claim 事务会重新计算凭证，不能相信客户端上传的数量或 source id。

## 7. API

### 命途长卷

- `GET /api/fate-chronicle/current`
- `POST /api/fate-chronicle/attempts`
- `POST /api/fate-chronicle/results`
- `POST /api/fate-chronicle/rewards/:milestoneId/claim`
- `GET /api/fate-chronicle/ops/overview`

### 三证归卷

- `GET /api/weekly-archive/current`
- `POST /api/weekly-archive/rewards/foundation/claim`
- `GET /api/weekly-archive/ops/overview`

所有玩家写接口要求 JWT 和 route-bound `session-v2` HMAC。客户端不得上传随机种子、章节完成、分数、凭证数量、凭证来源或奖励金额。

## 8. 幂等与跨进程

- `mutationId` 相同且业务体相同返回原回执；改参复用返回 `409 mutation_reused`。
- `clientAttemptId` 绑定账号、轮换、章节和誓约，不能改参重用。
- run、receipt、attempt、result 具有唯一约束；同一权威回执只能投影一次。
- 奖励 claim、钱包余额、ledger、mutation 回执和业务事实必须在同一个 `BEGIN IMMEDIATE` 中提交。
- 进程内写队列只是减压手段，正确性必须依赖 SQLite 唯一约束与写事务。
- 跨进程同时发车、投影或领奖最终只能产生一个业务事实和一条经济流水。

## 9. 前端

- 主菜单增加“命途长卷”工具入口，直接进入可玩的长卷页面，不做营销落地页。
- 页面包含本周归卷状态、五类凭证、三章进度、章节起始誓约选择、章节中段处置誓约二选一、可恢复 run、领奖、加载/空/错/游客态。
- 战斗继续复用 `AuthoritativeRunPanel`，但 mode 固定为 `fate_chronicle`，发车必须经过长卷服务。
- 失败页明确显示“未扣次数，可重试”；完成页返回长卷并刷新章节解锁与归卷证据。
- 所有异步响应绑定当前账号；切号后旧响应不得覆盖新账号状态。
- 桌面和 390px 移动端都必须可完成选章、选誓约、战斗、结算、回到长卷和领奖。

## 10. 非目标

- 不在 S107 中把现有离线主线全部移植到服务端；本阶段交付的是第一条 backend-owned 主线篇章。
- 不删除旧本地 PVE、挑战、远征或存档兼容路径。
- 不新增聊天、公会、交易、赠礼、战力道具或付费优势。
- 不为命途长卷新增正式排行榜。
- 不为处置誓约新增永久属性、账号级战力或跨周成长。
- 不让三证归卷覆盖现有各模式自己的里程碑奖励。

## 11. 验收

- S111 不新增数据库迁移；现有 V12 数据、旧账号、存档、PVP、裂隙、社交和接力数据保持不变；
- 九条实际誓约场景可确定性重放，稳健/进取/处置路线都可完成且无首动秒杀；
- 顺序解锁、全誓、失败重试、72 小时恢复、跨周宽限、旧 v1 冻结周卷兼容和迟到回执均有测试；
- 直接创建 `fate_chronicle` 通用 run 被拒绝，伪造章节/分数/凭证字段被拒绝；
- 五类凭证只认服务端权威事件，同模式刷多次仍只算一证；
- 2/5 可领一次，3-5 不增加货币；并发领奖只有一条 ledger；
- Node 全门禁、跨进程测试、真实后端浏览器桌面/移动流程、release report 结构汇总和生产 smoke 全部通过；
- 部署前备份，部署后验证 health/schema、systemd、Nginx、公网前端和真实 API smoke。
