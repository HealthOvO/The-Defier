# The Defier《逆命者》彻底大版本升级策划案 V3

## 1. 版本定位

本案基于当前项目真实状态提出一次真正意义上的“大版本升级”，建议版本名定为：

> **v6.0《天命裂界》**

它的目标不是“再加一点内容”，而是把《逆命者》从一个内容量已相当可观的网页卡牌 Roguelike，升级为一个同时具备：

1. **更强单局身份感**
2. **更深章节决策感**
3. **更稳长线留存**
4. **更高扩展效率**
5. **更统一仙侠视觉语言**

的长期可迭代产品。

## 2. 当前版本基线

结合仓库现状，当前版本已经不是原型，而是具备完整骨架的成熟版本：

| 模块 | 当前状态 |
|------|----------|
| 角色 | `6` 名可玩角色 |
| 卡牌 | `178` 张 |
| 法宝 | `46` 件 |
| 事件 | `61` 个 |
| 敌人 | `76` 个 |
| 主要模式 | 主线、无尽、传承、成就、图鉴、异步 PvP |
| 已有中层系统 | 命环路径、流派共鸣、法宝套装、战场指令、无尽压力、赛季 PvP 奖励 |
| UI 基础 | 主菜单、图鉴、战斗 HUD、PVP 页面均已可用 |

这意味着本次升级不需要“从零发明玩法”，而应在已有基础上进行**骨架升级 + 内容放大 + 体验收束**。

## 3. 当前版本的核心问题

虽然内容体量已经不小，但如果要继续提升可玩性，当前最需要解决的是以下 6 个结构性问题：

### 3.1 单局构筑仍偏“拿强卡通关”

已有流派和法宝很多，但很多局中后期仍容易收敛成：

- 拿高质量卡
- 拿泛用法宝
- 按常规路线推进

玩家能赢，但未必能强烈感到“这一把和上一把完全不是同一条命”。

### 3.2 中期节点有内容，但缺少“转折”

地图、营地、商店、事件大多承担“分发资源”的功能，较少承担：

- 改写整局打法
- 迫使玩家做长期取舍
- 引发高风险高收益的命运抉择

### 3.3 战斗深度已有，但章节生态还不够鲜明

当前已经有环境、精英、Boss、战场指令、敌方行为差异，但章节之间的“世界规则变化感”还可以更强。玩家打到新章节时，应该立刻感受到：

- 出牌节奏变了
- 资源压力变了
- 敌人威胁类型变了
- 事件叙事语气变了

### 3.4 局外成长有了，但长期目标还不够“层层递进”

传承、成就、PVP、图鉴已经具备，但玩家需要更明确的中短期目标：

- 今天上线要做什么
- 这周冲什么
- 这个赛季追什么
- 失败后我到底获得了什么新的可能性

### 3.5 界面风格已成型，但信息层级还可继续收束

当前水墨暗金风格已经有辨识度，但随着系统增多，部分页面出现“内容都重要，所以看起来都一样重要”的问题。尤其是：

- 主菜单入口层级
- 战斗中的关键状态优先级
- 命环 / 法则 / 共鸣 / 词缀 / 套装的视觉语法

### 3.6 后续新增内容的成本正在升高

仓库里已经能看到大量自动化测试和架构收口工作，这说明当前团队已经在主动对抗复杂度。下一次升级必须继续朝“可生产”方向设计，而不是单纯加更多耦合点。

## 4. V6.0 总目标

本次升级要把《逆命者》明确升级为：

> **以“命运构筑”为核心、以“章节世界规则”为中层变化、以“洞府长线目标 + 赛季挑战”为长期驱动的东方仙侠卡牌 Roguelike。**

### 4.1 核心体验目标

玩家在每一局里都能清楚回答 4 个问题：

1. **我是谁**
   - 角色、道途、命格、誓约、灵契共同定义身份。
2. **我靠什么赢**
   - 构筑主线和资源转换路径在前中期就逐步清晰。
3. **我在冒什么险**
   - 高收益一定有代价，风险来源和反制方法明确可读。
4. **我为什么还想再来一局**
   - 每次失败都在解锁新的路线，而不是只给一些永久数值。

### 4.2 版本成功标准

- 玩家单局的“构筑复述率”明显提升。
- 主线章节记忆点增强，不再是纯数值推进。
- 无尽、PVP、挑战、洞府形成日常和周常留存闭环。
- 新内容能以模板化方式持续新增，而不是每次都大改底层。

## 5. 设计支柱

### 支柱 A：一局一命

每一局都要有可以被玩家复述的命运故事，例如：

- “这把我靠噬业誓约压血养爆发，后面拿到血月法宝直接成型。”
- “这把我走了灵契护道流，靠阵灵替我拖过前两幕 Boss。”
- “这把我在第三章签了窥天誓，整局商店都很贵，但法则编织特别强。”

### 支柱 B：更多系统，但更容易读懂

新系统可以多，但信息表达必须更清晰。复杂性要尽量放在：

- 章节层决策
- 卡牌 synergies
- 资源转化

而不是放在“UI 看不懂”和“触发条件记不住”上。

### 支柱 C：强力一定有成本

遵循卡牌设计红线：

- 不做无条件 `0` 费高价值牌
- 不做无限制万能检索
- 不做即时胜利条件
- 抽牌引擎必须有限制
- 大幅降费和资源倍增必须设上限

### 支柱 D：短循环爽，中循环变，长循环稳

- **短循环**：回合内连段、爆发、斩杀、读敌意图
- **中循环**：章节机制、地图路由、誓约转折、流派成型
- **长循环**：洞府、传承、图鉴、赛季、天道榜、周挑战

### 支柱 E：所有新内容都为后续版本做容器

本次升级新增的每一个系统，都要顺便成为后续版本继续扩内容的“槽位”。

## 6. 大版本核心卖点

建议把 V6.0 的对外核心卖点收敛成 5 条：

1. **命运构筑 3.0**
   - 角色、命格、誓约、法则、法宝不再是平铺系统，而是多轴联动。
2. **章节世界规则**
   - 每章都拥有独特的生态、地脉、事件群和 Boss 语法。
3. **灵契护道**
   - 新增可收集、可成长、可战斗释放的灵契同伴系统。
4. **洞府长线**
   - 新增偏解锁型的局外成长目标，解决“打完这一把然后呢”的问题。
5. **赛季与挑战闭环**
   - 主线、无尽、PVP、周挑战不再割裂，而是围绕同一赛季主题联动。

## 7. 核心玩法 3.0：从“牌组成长”升级为“命运构筑”

### 7.1 新的单局四层结构

建议每一局由以下 4 层共同塑造：

1. **角色**
   - 提供基础卡组、主动技、属性、专属起手体验。
2. **命格**
   - 提供该局的规则修正器，决定打法倾向。
3. **誓约**
   - 提供高风险高收益的中期转折。
4. **法则 / 法宝 / 灵契**
   - 提供具体的战斗引擎和爆发方式。

这样玩家不是“拿什么算什么”，而是从开局起就逐步被引导进入一种独特命运。

### 7.2 新增系统：命格谱系 2.0

现有版本适合加入一个轻量但高辨识的开局系统：

- 开局从 `3` 个命格中选 `1`
- 每章结束可强化命格 `1` 次
- 命格不是单次 buff，而是局内规则修正器

命格分为 4 大类：

| 类型 | 作用 |
|------|------|
| 爆发型 | 首段增伤、斩杀窗口、连击回能 |
| 续航型 | 治疗转化、护盾留存、慢热成长 |
| 控制型 | 减益触发、干扰敌方行动、延后爆发 |
| 赌博型 | 自损换收益、延迟爆发、危机翻盘 |

首发建议制作 `24` 个命格，保证每个角色至少有 `4` 个高适配命格和若干泛用命格。

### 7.3 新增系统：逆命誓约 2.0

在每章末尾提供一次重大选择，不只是普通奖励，而是“改命”。

建议规则：

- 第 `1/3/5` 章末出现誓约机会
- 玩家整局最多持有 `2` 条誓约
- 同类誓约可升级，但不能无限叠

首发誓约方向：

| 誓约 | 收益 | 代价 |
|------|------|------|
| 焚命誓 | 爆发伤害、抽牌、终结能力上升 | 最大生命下降 |
| 镇狱誓 | 护盾、反击、控制能力上升 | 治疗效率下降 |
| 窥天誓 | 法则、事件、图鉴收益更高 | 商店变贵 |
| 噬业誓 | 积累业障换中后期收益 | 前中期稳定性下降 |
| 破界誓 | 战场指令与章节特殊机制更强 | 回合资源更紧 |
| 归寂誓 | 弃牌、消耗、献祭体系强化 | 手牌容错下降 |

目标是让每次进入中后期之前，玩家都要做一次“我要成为什么样的人”的决定。

### 7.4 新增系统：灵契护道

这是本次升级最有辨识度的新系统，建议作为新增核心卖点。

玩法定义：

- 每局可携带 `1` 个灵契
- 灵契有一个**常驻被动**和一个**蓄能主动**
- 灵契可在图鉴中收集、升阶、解锁皮肤与传记

灵契的作用不是“替代角色”，而是：

- 帮角色补短板
- 放大特定构筑
- 给失败后提供新的目标

首发建议 `8` 个灵契：

1. **霜螭**：冰霜控制、延后敌方高威胁回合
2. **烛鸦**：灼烧与献祭联动
3. **星狐**：抽牌、预知、牌序修正
4. **玄龟**：护盾留存、反击
5. **魇蝶**：减益扩散、脆弱状态放大
6. **灵猿**：连击、额外行动、灵力回复
7. **剑魄**：单点处决、破甲
8. **器灵**：法宝套装与装备触发强化

### 7.5 命环系统 2.0：路径树 + 碑文节点

当前命环系统已有 `destruction / wisdom / resonance / convergence` 等路径，建议升级为：

- **主路径**
  - 决定玩法身份
- **碑文节点**
  - 提供小型规则修正
- **跃迁节点**
  - 在关键等级提供一次大强化选择

命环成长建议如下：

| 命环等级 | 升级内容 |
|----------|----------|
| 1-3 | 解锁基础路径特性 |
| 4 | 第一次跃迁选择 |
| 5-6 | 解锁碑文槽位 |
| 7 | 第二次跃迁选择 |
| 8-9 | 提高路径联动上限 |
| 10 | 终极跃迁，决定最终局内身份 |

路径建议从当前 4 个扩展为 6 个：

- `convergence`：开局资源、首段伤害、流畅节奏
- `resonance`：技能连锁、抽牌、命格协同
- `destruction`：压血爆发、斩杀、风险收益
- `wisdom`：法则、事件、图鉴、临时牌控制
- `sanctum`：护盾、反制、减伤、阵御
- `calamity`：诅咒、业障、持续压制

### 7.6 法则系统 2.0：法则编织盘

现有法则适合从“单体被动”升级为“组合系统”。

建议改造为：

- `1` 个主法则槽
- `2` 个副法则槽
- `1` 个禁术槽

玩法价值：

- 主法则定义主体验
- 副法则提供支撑与反制
- 禁术提供高风险终局技

例子：

- 主法则选“火焰真意”，副法则挂“生命汲取”“时间回溯”，形成灼烧续航流
- 主法则选“空间裂隙”，副法则挂“剑意碎片”，形成低费穿透流

编织盘还可以和命环路径、誓约、灵契产生联动，形成高阶构筑空间。

### 7.7 法宝系统 2.0：炼器、器灵、套装深化

当前已有法宝与套装标签，下一步不建议只做“更多法宝”，而是做三层结构：

1. **基础法宝**
   - 提供明确的构筑方向
2. **形态件**
   - 改写法宝触发方式
3. **核心件**
   - 提供高稀有的流派上限

新增功能：

- **炼器重铸**
  - 改法宝副词条，不改基础身份
- **器灵灌注**
  - 让一件核心法宝获得额外互动
- **套装共鸣**
  - 2 件与 3 件效果分开设计

设计原则：

- 强度尽量做“机制变化”，不要只做“伤害 x2”
- 神话法宝继续限制为最多 `1` 件
- 套装效果避免无脑泛用，优先服务特定构筑

### 7.8 卡牌系统 2.0：更深，但不失控

本次大版本建议从 `178` 张卡扩充到 `230-240` 张规模。

新增内容结构建议：

| 内容 | 数量建议 |
|------|----------|
| 新 archetype 卡 | `36` 张 |
| 现有流派升级卡 | `18` 张 |
| 跨流派桥接卡 | `12` 张 |
| 中立功能卡 | `8-12` 张 |

#### 新增两套主流派

1. **咒契流 `cursebound`**
   - 自损、刻印、诅咒转收益、延迟处刑
2. **灵傀流 `soulforge`**
   - 召唤、装备、阵件、持续站场

#### 现有流派全面提纯

- `hemorrhage`：流血处决感更强
- `precision`：高操作连段更明确
- `entropy`：弃牌不再只是副系统，而是独立胜利路径
- `stormcraft`：多段与易伤窗口更清晰
- `vitalweave`：治疗转化更有爆发感
- `bulwark`：防守不只是拖回合，而是能主动压制

#### 卡牌设计红线

遵循 `card-game-design` 约束：

- 不做无条件 `0` 费核心卡
- 不做无限制万能检索
- 不做即时胜利牌
- 重复抽牌发动机必须“每回合一次”或绑定成本
- 降费效果必须带下限，禁止“费用降到 0”失控

### 7.9 战场系统 2.0：天象、地脉、阵面

当前环境系统已具备雏形，可以进一步升级为章节级“战场语法”。

每章由三类机制共同塑造：

1. **天象**
   - 战斗内动态变化，如血月、重力、雷暴、枯荣
2. **地脉**
   - 对该章所有战斗生效的规则，如护盾衰减、抽牌压缩、灼烧扩散
3. **阵面**
   - 影响敌我站场关系或前中后排交互

作用：

- 让章节差异体现在打法，而不是只体现在背景图
- 强化关卡记忆点
- 给角色和流派带来天然的强弱波动与规划空间

## 8. 角色与内容包升级

### 8.1 角色升级策略

本次版本不建议一口气做多个全新角色，而是采用：

> **“6 旧角色二阶分化 + 1 新角色压轴解锁”**

理由：

- 旧角色已有认知基础，二阶分化成本收益比高
- 新角色作为版本高潮，最适合拉动回流与讨论度

### 8.2 六名现有角色的二阶道途

| 角色 | 道途 A | 道途 B |
|------|--------|--------|
| 林风 | 逆势压血 | 绝斩处决 |
| 香叶 | 回生续航 | 蚀命诅咒 |
| 无欲 | 功德防反 | 业火献祭 |
| 严寒 | 解析规划 | 因果延迟 |
| 墨尘 | 星律连携 | 时序预埋 |
| 宁玄 | 灵器共鸣 | 镇阵壁垒 |

每个角色不改成硬职业，而是通过：

- 起手包
- 道途特性
- 推荐命格
- 专属誓约
- 灵契适配

形成双路线体验。

### 8.3 新角色提案：沈夜白

建议加入一名解锁角色，作为 V6.0 的大型内容亮点：

- **姓名**：沈夜白
- **称号**：禁契行者
- **定位**：高风险诅咒 / 召契混合型
- **特色**：能把诅咒、临时牌、消耗牌变成资源
- **适配流派**：`cursebound` / `soulforge`

他是最适合承接“誓约、禁术、灵契”三条新系统的人物，也最能体现大版本主题。

## 9. 章节、地图与关卡节奏重构

### 9.1 主线结构建议

建议将当前主线明确包装成：

> **6 章天域 × 每章 3 层关键节点**

这样既与当前“18 重天”概念兼容，又能显著提升章节感。

### 9.2 六大章节主题

| 章节 | 名称 | 主机制 | 主要情绪 |
|------|------|--------|----------|
| 第一章 | 碎誓外域 | 新手教会风险选择 | 试探、反抗 |
| 第二章 | 炉海天阙 | 资源灼烧、持续损耗 | 压迫、锻造 |
| 第三章 | 沉星古庭 | 预埋、连锁、星律地脉 | 神秘、计算 |
| 第四章 | 悬镜深渊 | 复制、幻像、诅咒反照 | 诡异、怀疑 |
| 第五章 | 血月禁庭 | 压血、献祭、狂化阈值 | 疯狂、赌命 |
| 第六章 | 终焉命庭 | 法则编织、终局问答 | 宿命、升华 |

### 9.3 地图节点从“功能点”升级为“决策点”

建议节点种类扩展为：

- 战斗
- 精英
- Boss
- 商店
- 营地
- 事件
- **观星台**：看未来奖励与章节词缀
- **炼器坊**：法宝重铸 / 器灵灌注
- **禁术坛**：换取高风险牌或誓约碎片
- **灵契窟**：获取或强化灵契
- **记忆裂隙**：观看章节短叙事并做剧情抉择
- **试炼碑**：自选难度加码，换高额奖励

### 9.4 章节节奏规则

结合 `level-design` 的节奏原则，每章需遵守：

1. **前段**
   - 安全展示该章新规则
2. **中段**
   - 用事件和小精英迫使玩家改变原打法
3. **末段**
   - 用 Boss 检验玩家是否真正理解本章机制

玩家不应只记得“第几层很难”，而应记得“那一章在和我玩什么”。

## 10. 敌人生态与 Boss 彻底升级

### 10.1 敌人按战术身份设计

建议所有敌人统一标注身份标签：

- `striker`：爆发输出
- `guardian`：保护与护盾
- `hexer`：减益与控制
- `supporter`：治疗、召唤、增益
- `disruptor`：干扰资源、打乱节奏

每章至少包含：

- `2` 组基础生态组合
- `1` 组高压精英组合
- `1` 个有鲜明语言的 Boss

### 10.2 Boss 统一升级为“三幕式”

每个 Boss 必须具备：

1. **可读的开场立场**
2. **中段转阶段**
3. **终局极招**

并且每一幕都要给玩家反制窗口，而不是纯数值飞升。

### 10.3 新增“破局条件”

Boss 不只是拼输出，还可设置破局目标，例如：

- 在 2 回合内打破镜面护盾
- 累积 3 次法则扰动打断 Boss 终式
- 先清理 2 个阵眼才能让本体吃满伤害

这会极大提高章节记忆点与操作感。

## 11. 事件、商店、营地、试炼全面升级

### 11.1 事件升级方向

当前已有 `61` 个事件，建议本次升级新增 `30` 个左右，并按章节和流派归组。

事件结构建议分成 4 类：

1. **构筑选择事件**
2. **剧情抉择事件**
3. **高风险契约事件**
4. **世界状态事件**

其中至少 `40%` 的新事件应能直接改写玩家本局的路线，而不是只给金币或回血。

### 11.2 商店升级为三页结构

| 页签 | 功能 |
|------|------|
| 基础页 | 稳定售卖卡牌、法宝、移除服务 |
| 契约页 | 高风险高回报商品，如禁术、诅咒、誓约碎片 |
| 传闻页 | 预告后续章节事件、Boss 倾向、稀有掉落方向 |

这样商店就不只是“买强度”，而是“花资源换信息、换方向、换风险”。

### 11.3 营地升级为四项操作

| 操作 | 作用 |
|------|------|
| 静养 | 恢复生命与异常 |
| 冥想 | 强化命格、法则、命环碑文 |
| 逆铸 | 重铸法宝 / 分叉升级卡牌 |
| 观星 | 预览下章环境并调整路线 |

### 11.4 试炼系统升级

建议把试炼改成“自选词缀难度”的高级节点：

- 玩家自选 1-2 个额外词缀
- 获得更高稀有奖励
- 周挑战可复用该系统

## 12. 长线成长：从“永久数值”升级为“永久解锁”

### 12.1 新增系统：洞府经营

这是本次大版本第二个最重要的新系统。

洞府不是数值农场，而是**偏解锁型元成长**。建议分为 4 个房间：

| 房间 | 功能 |
|------|------|
| 藏经阁 | 解锁图鉴页、构筑记录、推荐路线 |
| 炼器室 | 解锁法宝重铸、器灵灌注、套装研究 |
| 观星台 | 解锁周挑战、章节预兆、Seed 回放 |
| 伏魔台 | 解锁高难内容、Boss 记忆战、特殊皮肤 |

洞府能给玩家带来持续目标，但尽量避免“永久数值越来越大”的膨胀问题。

### 12.2 传承系统 2.0

传承建议继续以“解锁”和“起局差异”为主，而不是简单加面板。

传承奖励方向：

- 新命格池
- 新灵契
- 新事件分支
- 新法则编织位
- 新外观与称号
- 新起局卡组模板

### 12.3 命途悬赏

新增日常与周常：

- **今日天机**
  - 指定角色、指定章节词缀、指定奖励
- **七日劫数**
  - 一周内可重复挑战，追更高分
- **众生试炼**
  - 全服共通规则，挂接排行榜

这是最适合网页游戏长期运营的低成本高留存系统之一。

### 12.4 图鉴 2.0：观想藏经阁

图鉴不应只做“收集展示”，而应做成“理解工具”。

建议新增：

- 构筑回放
- 曾经通关的套牌快照
- Boss 克制信息
- 法宝 / 法则 / 命格联动建议
- 章节剧情碎片拼图

## 13. 无尽模式与 PVP 深化

### 13.1 无尽模式：从冲层改为“周期灾厄试炼”

当前无尽已具备压力、赐福、词缀、专属商店等骨架，V6.0 可进一步升级为：

- 每 `6` 层一个小轮回
- 每 `18` 层一个大轮回
- 每轮回刷新灾厄主题、商店方向、敌群构成、世界事件

新增无尽专属内容：

- 轮回偏执
- 极限赌命赐福
- 世界裂隙事件
- 周无尽榜

### 13.2 PvP：异步阵法对决 2.0

当前 PvP 已有段位、赛季、商店、奖励预估、称号与外观，下一步重点不是“做实时对战”，而是做更适合当前架构的异步深度。

建议升级点：

1. **赛季公共法则**
   - 每赛季改变全服 PvP meta
2. **防守阵灵**
   - 把灵契系统接入防守阵法
3. **镜像回放**
   - 回看本场战斗中的关键回合
4. **赛季道印**
   - 纯荣誉型进度线，强化展示欲
5. **论道合约**
   - 周赛制规则池，限制部分流派，刺激 meta 变化

## 14. 经济系统升级

结合 `economic-systems` 的稀缺与产消原则，建议采用三货币结构：

| 货币 | 来源 | 用途 |
|------|------|------|
| 灵石 | 局内战斗、事件、商店返利 | 局内购买与服务 |
| 传承精魄 | 通关、失败结算、周挑战 | 洞府与传承解锁 |
| 天机印 | 赛季、PVP、周无尽榜、特别活动 | 外观、称号、稀有图鉴奖励 |

### 14.1 经济设计原则

1. 局内货币只解决“这一局怎么玩”
2. 局外货币只解决“以后可以玩什么”
3. 荣誉货币只解决“我想展示什么”

这样可以避免不同系统互相污染。

### 14.2 产消平衡建议

- 商店价格随章节和风险动态调整
- 高价值服务绑定机会成本
- 周挑战奖励更偏外观、图鉴、解锁，不要过度发永久强度
- 失败也要有稳定进度，但胜利必须更高效

## 15. 美术、UI、动效、音频升级方向

### 15.1 视觉主题

建议把当前“水墨暗金”的视觉基础升级为统一的 **命纹仙侠语法**：

- 命环：圆环与裂痕
- 法则：符文、流线、元素色
- 誓约：封印、血印、契文
- 灵契：灵兽纹样、气脉光轨
- 章节：背景大色调与独特天象动画

### 15.2 信息架构重构

主菜单建议聚焦为：

1. 主线登天
2. 无尽轮回
3. 天道榜

二级入口集中到：

- 洞府
- 藏经阁
- 成就殿
- 设置与外观

### 15.3 战斗界面优先级

战斗中只让 3 类信息最抢眼：

1. **敌人这回合要做什么**
2. **我方当前能做成什么**
3. **本章 / 誓约 / 法则正在放大什么**

其余信息进入折叠层、抽屉层或悬停详情。

### 15.4 移动端要求

遵循 `ui-ux-pro-max` 原则：

- 关键按钮触控区不少于 `44x44`
- 关键字统一颜色，不靠纯文案堆信息
- 章节机制和战场词缀统一做成状态条，不要散落多个角落
- 高价值浮层必须可关闭、可返回、可在窄屏正常阅读

### 15.5 视觉落地建议

- 图标从 emoji 逐步替换为统一 SVG 体系
- 章节切换加入整页转场动画
- Boss 三幕切换加入专属 UI 断章感
- 灵契出场加入独立短动效
- 主要章节配乐增加 leitmotif，提升记忆点

## 17. 开发落地路线

### Phase 1：系统骨架版

目标：

- 命格
- 誓约
- 灵契
- 章节化主线结构
- 统一章节机制入口

验收：

- 任意一局都可完整走通“角色 -> 命格 -> 章节 -> 誓约 -> Boss -> 结算”

### Phase 2：内容放大量产版

目标：

- 新角色
- 两套新流派
- 新事件
- 新法宝
- 新章节敌人生态

验收：

- 新老角色都能走通至少 2 条不同构筑方向

### Phase 3：长线闭环版

目标：

- 洞府经营
- 周挑战
- 图鉴回放
- 无尽轮回升级
- PvP 赛季规则升级

验收：

- 玩家在失败、通关、挑战、PVP 之间形成稳定日常闭环

### Phase 4：体验抛光版

目标：

- UI 信息架构收束
- 美术与音频升级
- 新手引导与难度分层
- 移动端适配强化

验收：

- 新玩家可读性提升
- 老玩家高级系统理解成本下降

## 18. 平衡与验证方案

### 18.1 卡牌层

- 新卡默认不低于 `1` 费，除非效果极低且有限制
- 抽牌引擎统一设置次数或代价
- 检索效果统一做成“从若干选项中择一”，不做万能搜牌

### 18.2 构筑层

- 每个主 archetype 至少有 `1` 条清晰获胜路径
- 每个 archetype 至少有 `2-3` 张桥接卡防止锁死
- 高上限 build 必须带有明显弱点

### 18.3 章节层

- 每章都有“教规则 -> 变规则 -> 考规则”的完整节奏
- 环境机制不能只惩罚某一类 build 到不可玩

### 18.4 长线层

- 洞府和传承尽量偏解锁，不做数值膨胀
- 赛季与周挑战奖励优先给荣誉、外观、图鉴与玩法开关

### 18.5 自动化验证建议

建议新增或扩展以下门禁：

- `sanity_run_identity_checks`
- `sanity_vow_guardrail_checks`
- `sanity_spirit_companion_checks`
- `sanity_chapter_hazard_distribution_checks`
- `sanity_build_diversity_checks`
- `browser_meta_progression_audit`
- `browser_chapter_flow_audit`

## 19. 成功指标

### 可玩性指标

- 不同 archetype 的通关分布更均衡
- 玩家平均每局使用的核心系统数增加，但弃游率不上升
- 单局构筑复述率、截图分享率提高

### 留存指标

- 周挑战参与率提高
- 无尽与 PvP 的回访率提高
- 图鉴、洞府、外观系统的使用频次提高

### 内容生产指标

- 新事件、新法宝、新敌人可以模板化扩展
- 新增内容的自动化验证项可稳定接入
- UI 修改不再频繁牵动战斗核心逻辑

## 20. 最终建议

如果只用一句话概括这次升级方向，那就是：

> **把《逆命者》从“内容已经很多的仙侠卡牌肉鸽”，升级为“每一局都像一段命运传奇、每一章都像一场世界规则试炼、每一次失败都能带来新可能性”的长线构筑游戏。**

具体落地上，建议优先顺序为：

1. 先做 **命格 + 誓约 + 章节化主线**
2. 再做 **灵契 + 新角色 + 两套新流派**
3. 然后补齐 **洞府、周挑战、无尽 / PvP 赛季化**
4. 最后统一做 **UI、音频、美术与信息架构抛光**

这样既能在短期内立刻拉开版本差异，也能保证工程和内容生产都能跟得上。
