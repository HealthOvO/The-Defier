# The Defier《逆命者》V6.1《万象归途》功能需求文档 V4

## 1. 文档用途

本文是 [designer_major_upgrade_planning_v4.md](/Users/health/IdeaProjects/The-Defier/docs/designer_major_upgrade_planning_v4.md) 的配套需求文档。

本文不写排期、不写工时、不写人力，只回答一件事：

> **V6.1 到底要做什么，做到什么程度，页面和系统要补哪些能力。**

## 2. V6.1 必做范围

`V6.1` 至少要交付以下 `10` 大板块：

1. **裂界远征结构重做**
2. **章节悬赏系统**
3. **势力声望与敌意系统**
4. **仇敌追猎系统**
5. **构筑总览与命盘系统**
6. **章节结局分流**
7. **洞府推演所扩展**
8. **挑战 / 无尽 / PvP 联动升级**
9. **图鉴与奖励页决策化**
10. **模板化数据与门禁扩展**

缺任何一块，`V6.1` 的“可玩性与功能升级”都会不完整。

## 3. 玩家主流程需要升级成什么样

`V6.1` 后，主线局内流程需要演进为：

1. **开局准备**
   - 选角色
   - 选道途
   - 选命格
   - 选灵契
   - 查看本周赛季主题 / 挑战修正
2. **进入章节**
   - 查看章节天象、地脉、主线目标
   - 抽取 `3` 条章节悬赏并承接 `1-2` 条
   - 查看当前章节活跃势力与可达区域
3. **远征推进**
   - 在主线区域和支线区域间做取舍
   - 遭遇势力事件、商队、伏击、支线战斗
   - 决定是否追击仇敌或抢特殊奖励
4. **中局转折**
   - 签订誓约 / 升阶灵契 / 改写命格 / 调整路线
   - 根据悬赏和势力态度重新评估 build
5. **章节终局**
   - 面对章节 Boss 或特殊变体
   - 触发章节结局分流
   - 获得命盘评分、图鉴、洞府材料与模式联动奖励
6. **局外回环**
   - 洞府查看命盘、仇敌、势力、图鉴推进
   - 把该局命盘用于挑战 / 回放 / PvP 镜像 / 无尽演练

## 4. 裂界远征结构

### 4.1 要做什么

把当前章节地图从“节点分布图”升级为“主线 + 支线区域 + 动态目标”的远征结构。

### 4.2 功能要求

每章需要包含：

- `1` 条主线路径
- `2-3` 个支线区域主题
- `1` 个章节悬赏面板
- `1` 组活跃势力
- `1-2` 条仇敌线索
- `1` 个章节结局评估器

### 4.3 支线区域要求

每个支线区域都要有：

- 独立名称、视觉标签、叙事语气
- 更偏向的敌群 / 事件 / 商店 / 奖励
- 至少 `1` 条独有收益
- 至少 `1` 条明确代价或风险

支线区域示例：

| 区域 | 方向 | 风险 | 回报 |
|------|------|------|------|
| 焚骨矿场 | 爆发 / 伤害 | 敌方先手更强 | 高攻击奖励 |
| 星契遗府 | 法则 / 灵契 | 路线更长 | 高研究素材 |
| 禁录黑市 | 交易 / 赌局 | 势力敌意 | 稀有服务与法宝 |
| 封印外城 | 生存 / 防御 | 精英更密集 | 高护阵收益 |

### 4.4 UI 要求

地图页必须同时展示：

- 主线目标
- 支线区域入口
- 已承接悬赏
- 势力态度摘要
- 当前仇敌追猎状态

## 5. 章节悬赏系统

### 5.1 要做什么

新增章节悬赏，作为局中目标驱动器。

### 5.2 功能要求

- 每章开始时提供 `3` 选 `1-2`
- 部分事件与势力服务可追加 / 改写悬赏
- 悬赏完成度要实时可见
- 悬赏失败不会直接结束游戏，但会影响命盘评分与额外奖励

### 5.3 悬赏分类

必须支持至少四类：

- 战斗悬赏
- 路线悬赏
- 构筑悬赏
- 极限悬赏

### 5.4 文案和可读性要求

每条悬赏必须清楚写出：

- 完成条件
- 失败条件
- 预估奖励
- 更适配的 build 方向
- 是否与当前势力 / 仇敌 / 章节天象冲突

## 6. 势力声望与敌意

### 6.1 要做什么

新增势力层，使章节和事件真正拥有长期后果。

### 6.2 功能要求

每个活跃势力至少要有：

- 名称、图标、主张
- 好感 / 敌意状态
- 影响的节点类型
- 提供的援助
- 可能触发的惩罚
- 至少 `1` 条专属支线或结局钩子

### 6.3 系统行为要求

势力态度必须能影响：

- 商店价格或服务池
- 事件出现率
- 伏击与增援
- 仇敌归属
- Boss 开场状态或支援
- 章节结局描述

### 6.4 展示要求

地图页、事件页、章节结算页都要能看到：

- 当前倾向
- 最近一次态度变化原因
- 下一个态度阈值会触发什么

## 7. 仇敌追猎系统

### 7.1 要做什么

让部分精英与特殊敌人从“一次战斗对象”升级为“可追踪的对手”。

### 7.2 功能要求

首发应至少支持：

- `8-12` 名记名仇敌
- `3` 种成长路径
- `3` 种后续出现方式
- `1` 套图鉴追猎记录

### 7.3 仇敌行为要求

仇敌可发生：

- 直接击破
- 负伤逃逸
- 升阶复现
- 章节伏击
- 投靠势力
- 成为 Boss 护卫或特殊变体

### 7.4 奖励要求

追猎收益不应只有数值奖励，还应包含：

- 专属法则 / 法宝 / 事件线索
- 命盘评分加成
- 图鉴故事解锁
- 势力态度变化

## 8. 构筑总览与命盘系统

### 8.1 要做什么

新增一层系统总览，让玩家知道“我这局现在长成什么样了”。

### 8.2 构筑总览需求

需要输出以下信息：

- 当前 build 核心标签
- 五维成形度：
  - 进攻
  - 续航
  - 控制
  - 资源
  - 终结
- 当前短板
- 推荐补件方向
- 当前章节最危险的风险类型

### 8.3 命盘系统需求

每一局结束后生成一份命盘摘要，至少记录：

- 角色 / 命格 / 誓约 / 灵契 / 核心法则 / 核心法宝
- 章节结局
- 完成的悬赏
- 势力立场
- 击破或放走的仇敌
- 关键构筑标签
- 总评分与评分来源

### 8.4 命盘用途

命盘必须能用于：

- 洞府回看
- 挑战回放
- PvP 镜像展示
- 图鉴关联
- 赛季精选展示

## 9. 章节结局分流

### 9.1 要做什么

让“过章”有更多结果差异，而不只有是否胜利。

### 9.2 首发结局类型

每章至少支持以下结局标签中的若干：

- 强袭
- 结盟
- 追猎
- 封印
- 破局
- 残胜

### 9.3 结局影响要求

章节结局需要影响：

- 奖励池
- 势力态度
- 仇敌后续
- 图鉴文本
- 命盘评分
- 下章初始事件或章节提示

## 10. 洞府推演所扩展

### 10.1 要做什么

把洞府从“收集页 + 中枢页”升级为“分析、复盘、准备下一局的地方”。

### 10.2 必做房间

建议扩展为以下功能房间：

- 命盘档案室
- 仇敌追猎榜
- 势力关系馆
- 流派演算台
- 赛季陈列架

### 10.3 流派演算台要求

玩家应能看到：

- 当前已解锁内容下的推荐 build 组合
- 某套 build 缺少哪些法则 / 法宝 / 灵契
- 更适合在哪些章节、支线区域和势力路线练习

## 11. 挑战 / 无尽 / PvP 联动升级

### 11.1 今日天机 / 七日劫数

需要增加：

- 多目标挑战板
- 命盘回放入口
- 周进度奖励
- 与赛季修正器联动

### 11.2 无尽轮回

需要增加：

- 无尽专属悬赏
- 轮回奇景节点
- 周期性大灾变
- 更明确的赛季修正器试验标签

### 11.3 PvP

需要增加：

- 命盘摘要卡
- 命格 / 誓约 / 灵契展示
- 镜像试战
- 赛季特殊规则条目

## 12. 图鉴与奖励页决策化

### 12.1 图鉴要求

法则、灵契、敌影、Boss、法宝图鉴都要支持：

- 标签检索
- 适配 build 检索
- 章节 / 势力 / 仇敌关联
- 解锁来源追踪
- 推荐练习入口

### 12.2 奖励页要求

所有奖励卡片或服务条目应显示：

- 与当前 build 的匹配程度
- 它补的是哪项短板
- 它更偏即时收益还是长线收益
- 若选择它，可能放弃什么

## 13. 内容扩充要求

V6.1 首发内容至少应补齐：

- `1` 名角色或 `1` 条全局新路线
- `40+` 张卡牌
- `16+` 件法宝
- `25+` 个事件
- `18+` 个敌人与仇敌变体
- `6+` 个 Boss / 章节终局变体
- `4-6` 个支线区域主题
- `20+` 条悬赏模板
- `6+` 个势力模板
- `8+` 个仇敌模板

## 14. UI / 视觉要求

### 14.1 地图页

要新增：

- 章节导航板
- 势力态度条
- 支线区域入口标识
- 仇敌线索标识
- 活跃悬赏面板

### 14.2 战斗页

要新增：

- 当前悬赏进度提示
- 当前仇敌特殊机制提示
- 构筑总览快捷摘要
- 与当前战斗相关的势力 / 章节风险说明

### 14.3 结算页

要新增：

- 命盘评分拆解
- 本章结局说明
- 势力变化回顾
- 仇敌结果回顾
- 推荐下一步去向

## 15. 数据结构与自动化门禁

### 15.1 数据层必须新增

- `chapter_branch_regions`
- `chapter_bounties`
- `faction_profiles`
- `nemesis_profiles`
- `run_slate_templates`
- `chapter_endings`
- `season_modifier_templates`

### 15.2 自动化门禁必须新增

至少增加以下检查：

- 悬赏抽取、进度、结算
- 势力态度变化与阈值奖励
- 仇敌逃逸 / 复现 / 击破链
- 命盘生成与回放数据完整性
- 章节结局分流正确性
- 构筑总览输出的稳定性
- 地图与战斗 UI 中关键摘要的存在性

## 16. 最终结论

`V6.1《万象归途》` 的核心不是“再加更多系统”，而是：

- 把每一局变得更像旅程
- 把每次决策变得更有后果
- 把每个模式变得更互相需要
- 把复杂系统变得更容易判断

如果 `V6.0` 解决的是“让玩家玩得更像一条独特命”，那么 `V6.1` 要完成的是：

> **让每一条命，都真正走出不同的路，并且值得回头复盘。**
